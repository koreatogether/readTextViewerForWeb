# 📖 readTextViewer

> 창 크기에 맞춰 스크롤 없이 TXT 파일을 읽는 웹 뷰어

🔗 **[지금 바로 사용하기 →](https://koreatogether.github.io/readTextViewerForWeb/)**

---

## 만든 계기

전자책 리더나 PDF 뷰어는 많지만, 단순한 `.txt` 파일 하나를 쾌적하게 읽을 수 있는 도구가 없었습니다.

특히 소설이나 긴 텍스트를 읽을 때 불편한 점이 있었습니다:

- **와이드 모니터**에서는 한 줄이 너무 길어 눈이 피로하고, 좌우 페이지로 나눠보고 싶었습니다
- **세로로 세운 모니터**에서는 하나의 페이지를 꽉 채워 책처럼 읽고 싶었습니다
- 어디까지 읽었는지 기억이 안 나 매번 처음부터 스크롤해야 하는 번거로움이 있었습니다
- Windows 메모장에서 저장한 한글 파일이 다른 뷰어에서 깨지는 문제가 잦았습니다

이런 불편함을 해결하기 위해 **설치 없이 브라우저에서 바로 쓸 수 있는 뷰어**를 직접 만들었습니다.

## 특징

| 기능 | 설명 |
|------|------|
| 📐 **스크롤 없는 레이아웃** | 창 높이에 맞게 줄 수를 자동 계산 — 페이지가 화면을 꽉 채웁니다 |
| 🖥️ **양면 / 단면 자동 전환** | 가로 모니터는 좌우 2페이지, 세로 모니터는 1페이지로 자동 전환 |
| 🔤 **한글 인코딩 자동 감지** | UTF-16 LE/BE, UTF-8 BOM, UTF-8, EUC-KR(CP949) 순서로 감지 |
| 💾 **마지막 위치 자동 복원** | 브라우저를 닫았다 열어도 마지막으로 읽던 페이지에서 시작 |
| 📋 **최근 파일 목록** | 최근 열었던 파일 최대 10개 목록에서 바로 불러오기 |
| 🔖 **북마크** | 페이지에 이름을 붙여 저장하고 바로 이동 |
| 🎨 **테마 3종** | 화이트 / 다크 / 세피아 |
| 🔍 **텍스트 검색** | 키워드 검색 후 이전/다음 이동 |
| ⚙️ **설정 내보내기/불러오기** | 북마크, 최근 파일 목록, 설정을 JSON 파일로 백업/복원 |

## 사용법

### 파일 열기

**방법 1 — 버튼 클릭**
1. 상단 툴바의 `📂 파일 열기` 버튼 클릭
2. 탐색기에서 `.txt` 파일 선택

**방법 2 — 드래그 & 드롭**
- 탐색기에서 `.txt` 파일을 브라우저 창으로 끌어다 놓기

**방법 3 — 최근 파일 목록**
- 상단 `최근 항목` 드롭다운에서 이전에 열었던 파일 선택

---

### 페이지 이동

| 방법 | 동작 |
|------|------|
| `◀ 이전` / `다음 ▶` 버튼 | 한 페이지 이동 |
| 키보드 `←` / `→` | 페이지 위에 마우스를 올린 상태에서 이동 |
| 키보드 `Space` | 다음 페이지 (페이지 위에 마우스 올린 상태) |

---

### 보기 설정

| 설정 | 옵션 | 설명 |
|------|------|------|
| **글꼴 크기** | 12 ~ 32px 슬라이더 | 글자 크기 조절 |
| **줄 간격** | 1.0 ~ 3.0 슬라이더 | 줄 사이 간격 조절 |
| **보기 모드** | 자동 / 양면 / 단면 | 자동: 모니터 방향에 따라 자동 선택 |
| **테마** | 화이트 / 다크 / 세피아 | 배경과 글자색 변경 |

---

### 북마크

1. 기억해두고 싶은 페이지로 이동
2. 북마크명 입력 후 `추가` 클릭
3. 이후 드롭다운에서 선택하면 해당 페이지로 바로 이동

---

### 설정 백업 / 복원

- `설정 저장` → 북마크, 최근 파일 목록, 현재 설정을 `.json` 파일로 저장
- `설정 불러오기` → 저장해둔 `.json` 파일을 불러와 복원

## 지원 인코딩

Windows, macOS, Linux에서 저장된 한글 텍스트 파일 모두 지원합니다.

| 인코딩 | 설명 |
|--------|------|
| **UTF-16 LE** (BOM: `FF FE`) | Windows 10/11 메모장 기본 저장 방식 |
| **UTF-16 BE** (BOM: `FE FF`) | 일부 편집기 |
| **UTF-8 BOM** (BOM: `EF BB BF`) | 메모장 "UTF-8 BOM" 옵션 |
| **UTF-8** | 대부분의 현대 편집기 (VS Code 등) |
| **EUC-KR / CP949** | 구형 한국어 Windows 환경 |

## ⚠️ 주의사항

### 파일 내용이 브라우저에 저장됩니다

"마지막 위치 자동 복원"과 "최근 파일 목록" 기능을 위해 **읽은 파일의 내용이 브라우저의 로컬 저장소(localStorage)에 보관**됩니다.

- 저장된 데이터는 **해당 기기의 해당 브라우저에만** 존재하며 외부로 전송되지 않습니다
- 그러나 같은 기기를 여러 사람이 쓰거나 공용 PC를 사용할 경우, 타인이 브라우저 개발자 도구로 내용을 볼 수 있습니다
- **민감한 내용의 파일을 읽은 후에는 아래 방법으로 데이터를 삭제하세요**

```
브라우저 설정 → 개인정보 → 사이트 데이터 삭제
또는 개발자 도구(F12) → Application → Local Storage → 해당 항목 삭제
```

### 대용량 파일은 자동 복원이 안 될 수 있습니다

브라우저 localStorage 용량 한도(약 5MB)로 인해 **대용량 소설 파일**은 내용 자체가 저장되지 않을 수 있습니다.

이 경우 최근 파일 목록에 `⚠` 표시가 나타나며, 파일을 직접 다시 열어야 합니다.  
단, **마지막으로 읽은 페이지 번호는 별도로 저장**되므로, 같은 파일을 다시 열면 읽던 페이지에서 바로 시작합니다.

## 기술 스택

- **HTML5 / CSS3 / Vanilla JavaScript** — 프레임워크 없음, 설치 불필요
- **File API** — 로컬 파일을 브라우저에서 직접 읽기 (서버 업로드 없음)
- **localStorage** — 설정 및 세션 저장
- **TextDecoder API** — 다양한 인코딩 자동 감지
